SECTION 6: SPECIFIC TECHNICAL CONSTRAINTS
==========================================================================

MQL4 CONSTRAINTS:
- No standard C library (no sprintf, use DoubleToString)
- Custom functions: iATR, iRSI, iADX, iBollinger, iMA, iClose, etc.
- OrderSend() for trades, OrderModify() for changes
- Magic numbers identify strategies (777001-999001)
- _Point is symbol-specific point size
- Pips vs Points: 1 pip = 10 points on EURUSD (5-digit broker)
- Max lot size: 8.0, Min lot: 0.01
- Tester mode: Every tick (most accurate)

TELEGRAM CONSTRAINTS:
- No table syntax (use bullet lists or key:value)
- MEDIA: prefix for file delivery
- Supported: .png, .jpg, .webp (photos), .ogg (voice), .mp4 (video)
- NOT supported: .set, .mq4 as documents (must zip or use MEDIA: prefix)
- Markdown: **bold**, *italic*, ~~strike~~, ||spoiler||, `code`, ```blocks```

SERVER CONSTRAINTS:
- Linux Ubuntu on AWS EC2
- No GUI, no Wine (MT4 won't run)
- Python 3.11 available
- Git available
- Node.js available
- No MetaTrader compiler
